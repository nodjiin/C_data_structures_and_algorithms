Detailed Description

heap data structure.

Note
this data structure has been implemented as a min heap. However, since the priority comparison operation is passed as a function pointer, it is possible to convert it into a min heap by inverting the value returned by the custom implemented comparison.

related

construct a new heap.

Parameters
[in]initial_sizenumber of items that the heap will be able to contain after initialization. 0 is not considered a valid input, and will cause an early exit with INVALID_INPUT status code.
[in]comparecomparison function used to evaluate the priority of each item in the heap. An output > 0 will be considered as a sign that the first value passed to the function has lower priority than the second value, conversely with a value < 0 the first value will be considered having greater priority. If the output is 0 the 2 values will be considered of equal priority. NULL is not considered a valid input and will cause the program to exit early with an INVALID_INPUT status code.
Note
the actual size of the heap will always one unit bigger of the given initial_size. The additional space is used as offset as the beginning to simplify heap elements position calculations.
Returns
a pointer the newly constructed heap.
